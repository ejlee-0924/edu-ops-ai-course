# Claude Code vs Codex

둘 중 하나만 정답인 것은 아닙니다.
업무 성격에 맞춰 나누면 됩니다.

| 업무 상황 | 더 자연스러운 도구 | 이유 |
|---|---|---|
| 서비스 기획, 아이데이션 | Claude Code | 생각을 넓히고 말투를 다듬는 대화가 자연스럽습니다. |
| 사용자 여정, 운영 개선안 | Claude Code | 기획 파트너처럼 여러 선택지를 제안하기 좋습니다. |
| 이미 정해진 업무 처리 | Codex | 자료를 확인하고 정해진 양식으로 결과물을 만들기 좋습니다. |
| 반복 업무 자동화 | Codex | 폴더, 템플릿, 결과물을 구조화하기 좋습니다. |

## 한 줄 기준

> 아이디어를 같이 떠올리고 다듬을 때는 Claude Code, 정해진 업무를 정확히 처리하고 산출물로 만들 때는 Codex.

## 이번 실습에서는

두 도구 모두 쓸 수 있게 구성했습니다.

- Claude Code와 Codex 모두 이 웹사이트의 프롬프트를 사용할 수 있습니다.
- 업무 기준은 `AI 업무 매뉴얼 만들기`에서 한 장으로 정리합니다.
- 도구별 파일 이름보다 중요한 것은 목표, 자료, 결과물 기준을 분명히 주는 것입니다.

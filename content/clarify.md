# Clarify로 내 업무 구조화하기

Clarify는 도구 이름이기 전에, AI에게 일을 맡기기 전의 정리 방식입니다.

이 페이지를 시작하기 전에 `설치하고 시작하기`에서 Clarify 설치 또는 사용 가능 여부를 확인하고, `MCP 설치하고 자료 연결하기`에서 업무 자료 연결을 먼저 끝냅니다.
Claude Code에서는 `/plugin marketplace add`와 `/plugin install clarify`를 사용하고, Codex에서는 marketplace 추가 후 Plugins 또는 Skills 목록에서 Clarify를 활성화할 수 있는지 확인합니다.

도구마다 조금 다르게 반응할 수 있습니다.
Claude Code는 `Clarify로 해줘`만 입력해도 질문형으로 잘 시작되는 편입니다.
Codex는 바로 정리문을 만들 때가 있으므로, 아래 프롬프트처럼 **질문형 대화로 먼저 진행해달라**고 명시합니다.

실제 업무 구조화는 여기서 시작합니다.
완성된 양식을 먼저 채우지 않습니다.
AI가 질문자가 되어 수강생의 실제 업무를 하나씩 묻고, 답변을 모아 업무 실행 의뢰서와 개인용 Skill 초안으로 바꿉니다.

## 먼저 고르는 업무

아래 중 하나를 고르거나, 내 업무를 직접 적습니다.

- 만족도 정성 의견 리포트
- 고객사별 할 일 정리
- 수강생 안내 메일 작성
- CS 답변 초안 작성
- 교육 종료 후 결과 공유 자료 만들기
- 매주 운영 현황 요약

## 질문으로 정리할 기준

“만족도 리포트 정리해줘”보다 중요한 것은 아래 기준을 하나씩 정하는 경험입니다.

- 목표
- 입력 자료
- 결과물 형태
- 분류 기준
- 원하는 말투
- 주의사항
- 개인정보 처리 기준
- 완료 기준

## Clarify 실행 프롬프트

```text
Clarify로 내 반복 업무를 구조화해줘.

나는 비개발 교육 운영자야.
내 실제 반복 업무 하나를 AI에게 맡기는 개인용 Skill로 만들고 싶어.

바로 결과물을 만들지 말고, 먼저 질문형 대화로 진행해줘.
필요한 질문을 한 번에 하나씩만 물어봐줘.
내가 답하면 그 답을 반영해서 다음 질문으로 넘어가줘.

질문이 끝나면 아래 결과물을 만들어줘:
1. 업무 실행 의뢰서
2. 필요한 업무 자료 목록
3. 첫 실행 프롬프트
4. 개인용 Skill 초안
5. 첫 실행 전에 확인해야 할 개인정보/민감정보 체크리스트

첫 질문은 "어떤 반복 업무를 줄이고 싶나요?"로 시작해줘.
```

## 실습 파일

`practice/prompts/02-clarify-request.md`를 사용합니다.

## 다음 단계

Clarify 결과물이 나오면 `만족도 리포트 만들기`, `기업별 할 일 정리하기`, `고객사 메일 초안 만들기` 중 내 업무에 맞는 첫 결과물을 만듭니다.
나만의 업무 레시피 보강하기 단계에서는 여기서 만든 Skill 초안을 실제 첫 결과물 기준으로 보강합니다.

# Satisfaction Reporter

## 언제 쓰나요

교육 종료 후 만족도 조사 정성 의견을 리포트로 정리해야 할 때 사용합니다.

## 필요한 자료

- 만족도 조사 원본
- 과정명과 차수
- 고객사 공유용인지 내부 공유용인지
- 기존 분류 기준이 있다면 해당 기준

## 처리 순서

1. 만족도 원본에서 개인정보와 민감정보를 확인한다.
2. 정성 의견을 기본 카테고리로 분류한다.
3. 카테고리별 개수를 센다.
4. 카테고리별 대표 의견을 고른다.
5. 긍정 포인트와 개선 포인트를 분리한다.
6. 운영진 제안 의견을 정중한 말투로 작성한다.
7. 강사님께 공유할 요약을 별도로 작성한다.
8. 확인 필요 사항을 마지막에 표시한다.

## 출력 양식

- 전체 요약
- 카테고리별 의견 개수
- 대표 의견
- 긍정 포인트
- 개선 포인트
- 운영진 제안 의견
- 강사용 리포트 요약
- 확인 필요 사항

## 주의사항

- 실제 의견에 없는 내용을 만들지 않는다.
- 수강생 실명, 이메일, 전화번호는 제거한다.
- 고객사 공유용 문장은 과장하지 않는다.
- 강사용 요약은 비난이 아니라 개선 참고 자료로 보이게 작성한다.

## 좋은 결과물 기준

- 운영자가 1분 안에 교육 만족도 흐름을 파악할 수 있다.
- 고객사에 공유할 내용과 내부 개선 내용이 구분되어 있다.
- 다음 차수에 바로 반영할 액션이 보인다.
